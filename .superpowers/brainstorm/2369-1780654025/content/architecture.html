全站重构：架构设计方案

基于你的需求：沉浸式阅读主题 + 玻璃拟态风格 + 响应式三栏/底部导航 + API全量重构 + 分支隔离

核心交互流程设计

1. 用户旅程重构

A

阅读驱动型架构

以"发现书 → 立即阅读"为核心路径。首页即书架，最近阅读置顶，推荐流融入书架，减少导航层级。登录后可直接进入上次阅读章节。

B

内容管理型架构

以"管理书库 → 选择阅读"为核心路径。强化书架管理、分类标签、收藏夹功能。适合拥有大量书籍的重度用户。

C

探索发现型架构

以"发现新书"为核心路径。首页强化推荐算法、排行榜、分类浏览。弱化书架管理，强化内容发现。

3. 后端 API 重构方向

D

GraphQL + REST 混合

核心阅读接口用 GraphQL 实现按需查询（章节内容、进度、推荐一次性获取），管理接口保持 REST。减少前端请求次数。

E

统一 REST v2

在 /api/v2/ 下重新设计所有接口，采用资源嵌套、批量操作、统一响应格式（data/meta/error）。保持 Django Ninja 技术栈。

F

CQRS 读写分离

读接口（查询/推荐/排行）和写接口（收藏/进度/爬虫）分离，读接口深度优化缓存和聚合，写接口保证一致性。
